Editor's Note

Three things hit me today. First, infrastructure variables are swinging benchmark scores more than the actual differences between frontier models. Anthropic's piece on this should terrify anyone using public leaderboards to make decisions — same model, different provider setup, wildly different results.

Second, everyone's solving session management at once. GPT-5.5's long-running task support, Anthropic's managed agents with stable interfaces, Google's Workspace semantic layer — we've moved past "can it generate code" to "can it maintain context across actual work sessions."

The AGENTS.md piece nailed something I've been thinking about. Good documentation isn't just helpful for agents — it's literally a model upgrade. Bad docs actively hurt performance. How much of what we attribute to "model capability" is actually just inconsistent prompting infrastructure?
